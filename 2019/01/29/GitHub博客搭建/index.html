GitHub如何搭建一个静态博客

#为什么使用GitHub
免费方便，不用花一分钱就可以搭建一个自由的个人博客 可以绑定自己的域名
*博客内容可以轻松发布

在开始一切之前，你必须已经准备好

  • GitHub入门 Git简单命令的使用
  • 安装nodejs，直接上中文官网，有教程
  • 使用nodejs安装hexo

三分钟急速github

  • 创建仓库，新建一个名为username.github.io的仓库
    $ git init E：//文件夹（自定义路径） #初始化，并创建 .git目录
    注册的邮箱一定要验证
    一个github账户只能创建一个可以直接使用域名访问的仓库
    网站所有代码,都是放在这个仓库
    默认的 xxx.github.io 来访问,绑定域名可以解析CNAME记录下xxx.github.io，然后到你的github项目根目录新建一个名为CNAME的文件（无后缀），
    里面填写你的域名，找到你的仓库设置中绑定即可，
    配置使用的git命令如下：
    *$git config

  • /etc/gitconfig

  • –system

  • ~/.gitconfig

  • ~/.config/git/config

  • –global

#无论在系统做任何事，Git都会使用

$git config –global user.name “kong”
$git config –global user.email kong@qq.com

$git config –global core.editor vim

$git config –list

#查看所有配置

$git help 命令

$ git init

#初始化，并创建 .git目录

$ git add xxx

#跟踪文件
$ commit -m ‘initial project version’

#提交

$ git clone https://

克隆仓库

$ git status
$ git status -s -short

#检查，查看文件状态

#未跟踪 ?? 暂存区 A 修改过 M(右) 修改过并放入暂存区 M(左)

.gitignore文件

#指定忽略

  • 初始化hexo
    Hexo是一个简单、快速、强大的基于 Github Pages 的博客发布工具，支持Markdown格式，有众多优秀插件和主题。
    官网： http://hexo.io github: https://github.com/hexojs/hexo
    $ npm install -g hexo 打开命令行，安装
    $ hexo init 文件夹 自动初始化，使用默认主题设置
    $ cd 文件夹
    $ hexo g # 生成 $ hexo s # 启动服务

####常用hexo命令
hexo new “postName” #新建文章

hexo new page “pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本

  • 提交代码，上传到github
    配置_config.yml中有关deploy的部分：
  • deploy:
    **repository: git@github.com:username/username.github.io.git
  • branch: master
    npm install hexo-deployer-git –save 提交要使用
    配置git config
    $git config –global user.name “xxxx” 用户名
    $git config –global user.email xxx@xxx.com 邮箱
    $ commit最后提交，就可以访问到博客了
    ##第一次使用md格式,页面排版有点水
